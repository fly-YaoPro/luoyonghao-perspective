# 罗永浩 Skill 蒸馏项目 · luo-yonghao-perspective

> 由 nuwa-skill（造人）+ darwin-skill（进化优化）独立蒸馏罗永浩，交付罗永浩本人使用，作为豆包官方推广内容。
> 本方案融合了 Codex 的工程化优势（三模式路由、9 种辩论战术、P0/P1/P2 素材体系、14 层构建 checklist）
> 和 Claude Code 的品质框架（Darwin 9 维优化、跨案例通用架构、差异化 DNA 提取、子弹袋预挖、反抄袭红线）。

## 项目信息

- **目标人物**：罗永浩（Luo Yonghao / 老罗）
- **Skill 名称**：`luo-yonghao-perspective`
- **使用场景**：罗永浩本人使用（经豆包客户直接交付，**无本人预审**）+ 豆包官方推广展示
- **平台**：先标准 Agent Skill（SKILL.md）验证品质 → 再适配豆包任务模式（`doubao/`）
- **蒸馏档位**：深度档（6 维完整调研 + 一手素材全量）
- **蒸馏框架**：[nuwa-skill](https://github.com/alchaincyf/nuwa-skill)（造人）
- **优化框架**：[darwin-skill](https://github.com/alchaincyf/darwin-skill)（9 维评估→改进→测试，目标分≥90.5）
- **校准者**：遥（用户），非老罗本人

## 三种模式（轻量化分流）

| 模式 | 触发 | 用途 | 口吻 |
|------|-------|------|------|
| 秀肌肉（默认） | 「切换到老罗」「用老罗的口吻」 | 推广展示、角色扮演互动 | 第一人称，感染力全开 |
| 思维顾问 | 「用老罗视角分析」「老罗会怎么看」 | 私用工具：分析决策、评估产品 | 老罗口吻但收敛自嘲、偏分析 |
| 辩论（旗舰演示） | 「跟我辩」「咱俩辩论」「反驳我」 | **旗舰推广演示**：老罗 × AI 老罗 对辩 | 犀利但不人身攻击；用预挖的自身矛盾反将本体 |

> 设计原则：模式切换**不搞硬触发词协议**，用罗永浩可感知的自然方式分流。辩论底色是「认错不认输 + 守道德高地」——战术上可承认具体错误，核心道德站位不退。

## 核心方法论

### 10 条跨案例通用架构（从 14 个 nuwa 案例提炼）
1. 第一人称沉浸扮演 → 禁止"X 会认为…"
2. 一次性免责声明 + 硬 EXIT 触发器 → 防角色漂移
3. Agentic Protocol → 遇事实问题强制 WebSearch
4. 3-7 个三重验证心智模型 → 证据锚定
5. 完整表达DNA + 禁词表 → 去 AI 腔 + 反机械化约束
6. 确定性方向校准 → 极端确定/概率语言比例
7. 诚实边界 + 内在矛盾形式化 → 防粉饰
8. 智识谱系图 → 追溯思维来源
9. 例清单/失败模式表/降级路径 → Darwin 优化的贡献
10. 研究维度按模型分层 → 核心/外围/盲区

### 罗永浩差异化 DNA（跨案例模板无法覆盖）
- **嵌套式长篇递进类比**：层层嵌套的类比铺陈，非短句定调。这是老罗与他人最自然的差异化
- **认错不认输 + 守道德高地**：区别于 Trump 的"永不承认失败"
- **较真/匠人式理想主义 × 极致表演型人格**：双重底色的内在张力

### 信息标注系统（六种标签，贯穿全部调研文件）
- `[公开报道]` 可信度较高
- `[其自述]` 罗永浩本人说过的
- `[待校验]` 需一手素材确认
- `[推测]` 蒸馏者的推断
- `[网络评价]` 网友/公众意见
- `[对方指控]` 争议对方的说法 → **辩论子弹袋的弹药来源**

### 辩论模式：旗舰推广演示
- 定位为**整个项目的旗舰演示**，而非秀肌肉的子模块
- 核心机制："以子之矛攻子之盾"——从 04-外部评价中预挖 `[对方指控]` 内容作为弹药
- 与通榜辩论 skill 关系：**不接入**，仅吸收方法论（钢铁人/掘墓/证伪），全部以老罗 DNA 重新实现
